## Project Overview

TravelHub is a travel booking platform designed to help users search destinations, explore travel packages, book trips and flight ticket online.
This project focuses on performing manual testing on the TravelHub application to ensure that its features work correctly, meet business requirements, and provide a smooth user experience.

The testing activities were managed using Jira following the Agile methodology with sprint-based execution.

## Testing Methodology

This project follows the Agile testing approach.
Testing activities were planned and executed in sprints, where each sprint included a set of user stories and bug tracking.

Workflow followed during testing:

1. Requirement analysis from user stories in Jira
2. Test planning
3. Test case design
4. Test execution
5. Bug reporting and tracking in Jira
6. Test closure and reporting

## Test Scope

Modules tested in this project include:
1. User Registration and Login and Logout
2. Room Reservation
3. Flight Ticket Booking
4. Attractions
5. Payment Module
6. Profile Management
7. Navigation and UI validation

## Defect Management

All defects discovered during testing were logged and tracked in Jira.
Each defect includes:
1. Bug summary
2. Steps to reproduce
3. Expected result
4.Actual result
5. Severity and priority
6. Environment

## Key Outcomes

1. Multiple functional issues were identified during test execution.
2. Bugs were documented and tracked through Jira for resolution.
3. Test documentation provides traceability between requirements and test casees.
